# Pong Server (Node.js + Socket.io)

โปรเจกต์นี้เป็นเซิร์ฟเวอร์สำหรับเกม Pong แบบ Multi-player ที่พัฒนาด้วย Node.js และ Socket.io โดยใช้การคำนวณสถานะของเกม (Game State) ทั้งหมดบนเซิร์ฟเวอร์ (Server-side Authority) เพื่อความแม่นยำและป้องกันการโกง

## คุณสมบัติเด่น
- **Real-time Synchronization**: ใช้ Socket.io ในการส่งข้อมูลสถานะเกมไปยังผู้เล่นทุกคนพร้อมกันที่ 60 FPS
- **Server-side Physics**: การคำนวณตำแหน่งลูกบอล การชนของไม้ตี (Collision) และการนับคะแนน เกิดขึ้นที่ฝั่งเซิร์ฟเวอร์
- **Player Roles**: รองรับผู้เล่น 2 คน (Player 1 และ Player 2) และผู้ชม (Spectator) ไม่จำกัดจำนวน
- **Auto-restart**: ระบบนับเวลาถอยหลัง 5 วินาทีเพื่อเริ่มเกมใหม่โดยอัตโนมัติเมื่อได้ผู้ชนะ

## โครงสร้างโปรเจกต์
```text
pong-server/
├── src/
│   ├── gameState.js      # จัดเก็บข้อมูลสถานะของเกม (ตำแหน่ง, คะแนน, ผู้เล่น)
│   ├── gameLoop.js       # ตรรกะการคำนวณฟิสิกส์ การชน และลูปหลักของเกม
│   ├── socketHandlers.js # จัดการการเชื่อมต่อและเหตุการณ์ต่างๆ ของ Socket
│   └── server.js         # ไฟล์หลักสำหรับเริ่มต้น Express และ Socket.io Server
├── package.json          # ไฟล์กำหนด Dependency และ Scripts
└── README.md             # เอกสารอธิบายโปรเจกต์
```

## เทคโนโลยีที่ใช้
- **Node.js**: สภาพแวดล้อมสำหรับรัน JavaScript ฝั่งเซิร์ฟเวอร์
- **Express**: เว็บเฟรมเวิร์กสำหรับจัดการ HTTP Request
- **Socket.io**: ไลบรารีสำหรับสื่อสารแบบ Real-time Bi-directional
- **CORS**: รองรับการเชื่อมต่อจาก Client ต่าง Origin (เช่น Angular ที่รันบนพอร์ต 4200)

## ข้อมูลสถานะเกม (Game State)
สถานะของเกมจะประกอบด้วย:
- `paddle1`, `paddle2`: ตำแหน่งและขนาดของไม้ตีทั้งสองฝั่ง
- `ball`: ตำแหน่ง (x, y), รัศมี และความเร็ว (dx, dy)
- `score1`, `score2`: คะแนนของผู้เล่น
- `status`: สถานะปัจจุบัน (`waiting`, `playing`, `gameover`)
- `winner`: ชื่อผู้ชนะเมื่อจบเกม
